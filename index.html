Portfolio

School projects

Weather station

The aim of this project was to create a weather station in C using an Arduino board. To do this, we had various elements such as electrical wires, a GPS beacon and various sensors. This data was to be automatically recorded on an SD card integrated into the Arduino.

Commercial management application

The aim of this project was to develop a C++ application for managing various elements of a business, such as stocks, orders and customers, as well as producing statistics based on various parameters.

Document backup application

The aim of this project was to develop a document backup application. During a backup, each action was recorded in a file and all the elements saved were automatically encrypted.

Setting up a city-wide network

The aim of this project was to set up a secure computer network in a fictitious town using Cisco's Packet Tracer software.

A complete infrastructure

The aim of this project was to learn how to set up a complete IT infrastructure within a company. This involved knowing how to set up an Active Directory server, how to secure confidential documents and how to block access to the network from the Internet using firewalls.

A website for students

The aim of this project was to develop a website using HTML, PHP, CSS and JavaScript. The aim of this fictitious site was to allow companies to post internship offers themselves on a site to which students had easy access. All access had to be filtered via user accounts and the data contained in a secure PHP database.

Internship & fixed-term contract projects

Powershell opportunities

The PowerShell scripting language was mainly useful to me during my placement at the Lorraine cancer institute. I used it to create several scripts that interacted with the hospital's Active Directory system in order to check that the user groups did not contain any errors that could have compromised the security of the hospital's confidential documents.

A powershell robot

The aim of this project was to develop a robot that performed actions identical to those of a user on an application. This robot recorded the loading time of the various menus in the application and then sent its data to an Oracle database. The Lorraine Cancer Institute's ServiceNav supervision software could then use this data to produce application performance graphs.

Setting up and configuring ServiceNav and Zabbix

During my previous placements, I had the opportunity to implement two monitoring solutions: Zabbix and ServiceNav. These solutions retrieve performance data from the various IT servers and then produce graphs to inform us of changes in their status. In the event of problems, an alert is sent to warn the user that certain actions need to be taken on the servers before more serious problems occur.

Improving IT security rules

The aim of this project was to identify flaws in the security system at the Lorraine Cancer Institute. To improve the system, I reduced the time before user passwords expired to 90 days for all users. This project also included scripted verification of user access so that only authorised users could access reserved documents.

ESXi & VCenter

The aim of this project was to set up 2 completely empty servers running an ESXi operating system. For this project, I first had to upgrade the physical components of the servers, which were very old. I then set up the complete infrastructure that is now in use at DEFI Informatique.

Setting up applications using docker

The main advantage of containerising applications is the resulting performance gain. During this project, I set up two virtual machines running Debian 11, each hosting a Portainer server. Their respective roles were to host the containerised applications, in particular GLPI and Bookstack, and to host a NGINX reverse proxy that would enable secure access from outside the company.
